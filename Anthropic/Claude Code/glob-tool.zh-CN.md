＃ 这`Glob`工具

一个快速的**文件名/路径匹配**工具。它回答了以下问题：*“存在哪些文件
路径与此模式匹配吗？"*

Glob 搜索文件**名称和路径** — 它**不**查看文件内部。 （它的兄弟姐妹，`Grep`，搜索文件**内容**。看`grep-tool.md`.)

---

## 何时使用

- 按名称或扩展名查找文件（例如所有`*.test.ts`文件）。
- 在深入研究之前发现代码库的结构/布局。
- 查找与模式匹配的最近修改的文件。
- 任何时候你伸手去拿`find . -name ...`或终端中的 shell 通配符。

> **重要：** 优先使用此工具而不是运行`find`或者`ls`通过Bash文件工具
> 发现。它速度更快，适用于任何代码库大小，并返回有用排序的结果。

---

## 参数

|参数|类型 |必填 |描述 |
| --------- | ------ | -------- | -------------------------------------------------------------------------------------------------------- |
|`pattern`|字符串| **是** |用于匹配文件的 glob 模式（例如`**/*.js`, `src/**/*.{ts,tsx}`).               |
| `path`|字符串|没有 |要在其中搜索的目录。默认为当前工作目录。 **禁止通过`undefined`或者`null`表示“当前目录”——只需省略该参数即可。 |

---

## Glob 模式语法

Glob 模式比正则表达式更简单。最有用的代币：

|代币|意义|
| ----------- | ----------------------------------------------------------------------------------- |
|`*`|匹配任何一串字符**除了**路径分隔符（`/`).        |
| `**`|递归地匹配任意数量的目录（包括零）。        |
|`?`|恰好匹配一个字符。                                          |
|`{a,b,c}`|匹配任何逗号分隔的替代项（大括号扩展）。      |
|`[abc]`|匹配集合中的一个字符。                                     |

### 常见模式

-`*.md`— 搜索目录中的每个 Markdown 文件（仅限顶层）。
-`**/*.md`— **任何**深度的每个 Markdown 文件。
-`src/**/*.js`- 全部`.js`下任意位置的文件`src/`.
- `**/*.{ts,tsx}`- 全部TypeScript和 TSX 文件，递归地。
-`**/test_*.py`- 全部Python以以下内容开头的文件`test_`，在任意深度。
-`?[0-9]*.log`— log files whose second character is a digit.

---

## 输出

- 返回**匹配文件路径的列表**。
-结果**按修改时间**排序，最近修改的排在最前面。这很方便：
  您刚刚接触过的文件（或与最近工作最相关的文件）往往会出现在
  顶部。
- 如果没有匹配的内容，则返回空结果（不是错误）。

---

## 示例

**项目中任意位置的所有 TypeScript 文件：**```json
{ "pattern": "**/*.ts" }
```**测试特定目录下的文件：**```json
{
  "pattern": "**/*.test.{ts,tsx}",
  "path": "packages/web/src"
}
```**存储库根目录下的配置文件：**```json
{ "pattern": "*.{json,yml,yaml,toml}" }
```**每个自述文件，无论深度如何：**```json
{ "pattern": "**/README.md" }
```---

## Glob 与 Grep — 哪一个？

|你想要…… |使用 |
| ------------------------------------------------- | -------- |
|以某种方式查找 **命名** 的文件 | `Glob` |
|查找**包含**某些文本的文件 | `Grep` |
|列出所有 `*.py` 文件 | `Glob` |
|查找哪些 `*.py` 文件调用 `requests.get` | `Grep`（与 `glob: "*.py"`）|
|了解目录布局 | `Glob` |
|阅读匹配的代码行 | `Grep` (`output_mode: "content"`) |

常见的工作流程是**链接它们**：使用 `Glob` 缩小到一组候选文件，
然后 `Grep` 来搜索其内容。请注意，`Grep` 也接受其自己的 `glob` 参数，
因此，对于简单的“搜索这些文件类型”的情况，您可能根本不需要单独的 `Glob` 调用。

---

## 提示和陷阱

- **`*` 不跨越目录边界； `**` 确实如此。** `src/*.js` 直接匹配文件
  `src/`，而 `src/**/*.js` 匹配 `src/` 和每个子目录中的 `.js` 文件。
- **在整个项目中省略 `path`。** 不要传递空字符串或占位符 - 只是
  将其保留以从工作目录中搜索。
- **按新近度排序。** 因为结果是按修改时间排序的，所以 `Glob` 是一个快速
  回答“我最近更改了什么与 X 匹配？”的方法
- **对于跨许多目录和命名约定的广泛、开放式搜索**，请考虑
  委托给勘探代理，而不是发出许多单独的 `Glob` 调用。
