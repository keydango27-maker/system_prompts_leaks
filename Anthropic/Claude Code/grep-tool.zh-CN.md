# `Grep` 工具

建立在 [ripgrep](https://github.com/BurntSushi/ripgrep) 之上的强大的**内容搜索**工具
(`rg`)。它回答了以下问题：*“哪些文件包含与此模式匹配的文本，以及什么
是匹配的线吗？"*

Grep 搜索**内部**文件。 （它的同级 `Glob`，搜索文件 **名称**。请参阅
`glob-tool.md`。）

---

## 何时使用

- 查找定义或使用函数、变量、类或字符串的位置。
- 定位代码库中所有出现的模式。
- 计算某事物出现的次数。
- 任何时候您在 shell 中获取 `grep`、`rg`、`egrep` 或 `grep -r` 时。

> **重要提示：** 始终优先使用此工具，而不是通过 Bash 工具运行 `grep`/`rg`。
> 它是专门构建的，默认情况下遵循 `.gitignore`，并以干净、
> 结构化形式。不鼓励在 Bash 中运行 `grep`。

---

## 参数

|参数|类型 |必填|描述 |
| -------------- | -------- | -------- | ------------------------------------------------------------------------------------------------------------------ |
| `pattern` |字符串| **是** |要搜索的正则表达式。支持完整的正则表达式语法（例如 `log.*Error`、`function\s+\w+`）。 |
| `path` |字符串|没有 |要搜索的文件或目录。默认为当前工作目录。                          |
| `glob` |字符串|没有 |用于过滤搜索文件的全局模式（例如 `*.js`、`*.{ts,tsx}`）。映射到 ripgrep 的 `--glob`。 |
| `type` |字符串|没有 |要搜索的文件类型（例如 `js`、`py`、`rust`、`go`）。对于标准类型，通常比 `glob` 更高效。 |
| `output_mode` |字符串|没有 | `content`、`files_with_matches`（默认）或 `count` 之一。控制返回的内容（见下文）。 |
| `-i` |布尔 |没有 |不区分大小写的搜索。                                                                            |
| `-n` |布尔 |没有 |显示行号。仅当 `output_mode` 为 `content` 时适用。                                    |
| `-A` |数量 |没有 |每场比赛结束后**显示的上下文行。 （仅限 `content` 模式。）
| `-B` |数量 |没有 |每场比赛之前显示 **B** 的上下文行。 （仅限 `content` 模式。）
| `-C` |数量 |没有 |每场比赛之前 ** 和 ** 之后显示的上下文行。 （仅限 `content` 模式。）
|`multiline` |布尔 |没有 |启用多行模式，以便 `.` 匹配换行符，并且模式可以跨行。默认 `false`。         |
| `head_limit` |数量 |没有 |将输出限制为前 N 行/条目（例如通过管道传输到 `head -N`）。适用于所有输出模式。 |

---

## 输出模式解释

`output_mode` 参数更改了 get 的“内容”。选择正确的可以保持结果
重点：

- **`files_with_matches`** *（默认）* — 仅返回包含以下内容的文件路径列表
  至少一场比赛。当您只需要知道某物“在哪里”时最好。最便宜的输出。

- **`content`** — 返回实际匹配行（以及可选的周围上下文
  `-A`/`-B`/`-C`）。该模式支持 `-n`、`-A`、`-B` 和 `-C`。当你
  需要“读取”匹配项，而不仅仅是找到文件。

- **`count`** — 返回每个文件的匹配数。用它来衡量一个
  模式是在潜入之前。

---

## 正则表达式注释

- `pattern` 是正则表达式，**不是**文字字符串。字符如 `.`、`(`、
  `)`、`{`、`[`、`*`、`+`、`?`、 `|`、`\` 有特殊含义。
- 要按字面意思匹配它们，请使用反斜杠转义。示例：查找文字
  `interface{}` 在 Go 中，将模式写为 `interface\{\}`。
- Ripgrep 使用 Rust 正则表达式引擎。默认情况下，模式**每行**匹配 - a
  除非设置了 `multiline: true`，否则模式不能跨越多行。

## 示例

**查找提到 `TODO` 的每个文件（仅文件列表）：**```json
{ "pattern": "TODO" }
```**显示带有行号的匹配行，仅在 JavaScript 文件中：**```json
{
  "pattern": "useState",
  "glob": "*.js",
  "output_mode": "content",
  "-n": true
}
```**不区分大小写的搜索，包含 3 行周围上下文：**```json
{
  "pattern": "deprecated",
  "output_mode": "content",
  "-i": true,
  "-C": 3
}
```**计算每个文件中 `console.log` 在 TypeScript 源中出现的频率：**```json
{
  "pattern": "console\\.log",
  "type": "ts",
  "output_mode": "count"
}
```**多行搜索 - 跨越多行的 `struct` 块：**```json
{
  "pattern": "struct \\{[\\s\\S]*?name",
  "multiline": true,
  "output_mode": "content"
}
```---

## 提示和陷阱

- **尽早过滤。** 将 `pattern` 与 `glob` 或 `type` 组合起来比
  搜索一切并排除噪音。
- **`type` 与 `glob`。** `type` 使用 ripgrep 的内置语言定义（它知道哪个）
  扩展名属于 `python`、`rust` 等）。 `glob` 让您精确控制文件
  模式。使用更清楚地表达您的意图的那个。
- **在 JSON 中转义。** 因为模式通过 JSON，所以正则表达式中的反斜杠必须是
  在 JSON 字符串中写为 `\\`（例如 `\\.` 以匹配文字点）。
- **`.gitignore` 感知。** 与 ripgrep 一样，该工具会跳过忽略的文件（例如 `node_modules`）
  默认值——通常是你想要的。
- **为了探索，请与 `Glob` 配对。** 使用 `Glob` 按名称发现候选文件，然后
  `Grep` 来搜索其内容 - 或将两者都交给代理进行广泛的扇出搜索。
